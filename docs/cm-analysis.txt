Content Manager

Сервис должен уметь хранить контент, его тип, размер, хэш.

Должны быть реализованы api для добавления, получения, удаления контента.

Должно поддерживаться отсутствие дубликатов контента,
это должно происходить незаметно для клиента сервиса.

Ключи, по которым клиент может получить доступ к контенту, не должны меняться.

Ключем для доступа к контенту является его хэш.

Для реализации используется:
    1. Spring Boot
    2. Spring Data JPA
    3. h2 database
    4. Hibernate

Для получения хэшей контента используется алгоритм Sha256
